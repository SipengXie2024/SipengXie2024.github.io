        TROUBLESHOOTING INDEX

        按症状定位证据链

        症状先检查常见原因
        模型无法加载文件大小、格式、运行时架构支持下载不完整、量化不支持
        系统严重换页权重 RSS、KV、系统余量容量只按文件大小计算
        首 token 很慢冷启动、预填速度、上下文长度未预热、提示过长、磁盘 KV
        页面能打开但权重极慢重定向后的 CDN 路由所有 Hugging Face 域名走同一代理
        API 404完整请求路径Base URL 多一层或少一层
        Claude Code 要求登录AUTH_TOKEN 与配置来源配置未加载或被其他设置覆盖
        Claude Code 提前压缩客户端识别的上下文未设置自定义模型窗口
        只能聊天不能编码工具调用测试后端未兼容 Anthropic tool_use
        PID 存在但请求失败端口和健康接口服务卡死、错误监听地址

        FIVE WORKING PRINCIPLES

        真正可迁移的是方法，不是某一组命令

        1. 01

          用实际制品和实测日志，替代对参数量的想象。

        2. 02

          把上下文能力拆成模型、运行时、KV 与客户端四层。

        3. 03

          按真实下载链路分流，不做粗暴的全局直连或代理。

        4. 04

          把服务验收推进到真实工具调用，不停在 PID 或聊天回复。

        5. 05

          配置必须可启动、可观察、可回滚，而非一次性终端会话。
