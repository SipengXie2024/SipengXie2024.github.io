01

Research

My research treats blockchain performance as a full-stack problem, from transaction scheduling and execution engines to authenticated data structures, with a parallel focus on trusted AI relays.

Scheduling layer NSDI '27 / Under review

Octopus

Heuristic scheduling and multi-version cache orchestration make unordered parallel transaction execution deterministic while outperforming advanced baselines.

2-4xexecution speedup

Execution engine EuroSys '27 / Under review

SkelOT

Contract skeletons become reusable AOT compilation units, halving compilation overhead. A storage footprint of only 0.8% covers 75% of hot execution time in the evaluated blocks.

50%lower compilation overhead

Data layer USENIX Security '26 / Accepted

MHOT

A height-optimal trie turns the exponential span-fanout coupling into a linear one, replacing MPT while fully resisting the Nurgle attack.

9xwrite throughput, 4x less write amplification, 2x smaller commitments

Trusted relay IEEE TIFS '26 / Under review

AEGIS

A minimal TCB keeps the relay's critical forwarding logic inside a TEE, defending against leakage, prompt injection, and response tampering while enabling verifiable faithful forwarding.

TEEattested end-to-end relay integrity
